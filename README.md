# Solar Business Performance — Sales & Pipeline Dashboard

แอปบันทึก/ติดตามโครงการที่กำลังเสนอขาย (Sales Pipeline) สำหรับงานขาย Solar / Battery
ไฟล์เดียวจบ เปิดได้ทั้งบนคอมและมือถือ บันทึกข้อมูลลงเครื่อง + ซิงก์ Google Sheet ได้

## ไฟล์ในโฟลเดอร์
- `index.html` — ตัวแอป (เปิดด้วยเบราว์เซอร์ได้เลย หรือ host ขึ้น GitHub Pages)
- `customers.js` — ฐานลูกค้า 4,768 ราย ที่แอปโหลดมาทำ autocomplete (**ต้องวางคู่กับ index.html เสมอ**)
- `sales_history.js` — ข้อมูลยอดขายย้อนหลัง 2559–2569 สำหรับกราฟและ Drill-down
- `pi_items.js` — รายการสินค้าในแต่ละ PI สร้างจากไฟล์ Report_sale
- `สรุปยอดขาย 2559-2569  สรุปยอดขาย.xlsx` — ไฟล์ต้นทางยอดขายย้อนหลัง
- `scripts/build_sales_history.py` — สร้าง `sales_history.js` ใหม่เมื่อแก้ไข/เพิ่มชีตปีใหม่
- `scripts/build_pi_items.py` — สร้าง `pi_items.js` ใหม่เมื่อเปลี่ยนไฟล์ Report_sale
- `AppsScript.gs` — โค้ดหลังบ้านสำหรับซิงก์ Google Sheet
- `clean_customers.py` — สคริปต์ล้างฐานลูกค้าจาก sheet001.html (รันซ้ำได้เมื่อมี export ใหม่)
- `README.md` — ไฟล์นี้

## นำเข้าใบเสนอราคา (Report_sale) — สำคัญที่สุด
ไฟล์ `Report_sale_YYYYMMDD.xls` ที่ export จากโปรแกรมบริษัท = ดีลทั้งหมดของคุณพร้อมสถานะ
- กดปุ่ม **"นำเข้า PI"** ในแถบเครื่องมือ → เลือกไฟล์ .xls → แอปสร้าง/อัปเดตโครงการให้อัตโนมัติ
- map Status → สถานะ: `NEW PI`→เสนอราคา · `30%`→เสนอราคา · `50%/90%`→เจรจา · `PO WIN/PO IN`→ปิดได้ · `LOST`→หลุด (ใช้ % เป็นค่าโอกาสปิด)
- ลูกค้าลิงก์กับฐาน customers.js (เบอร์/TaxID/ที่อยู่) · สินค้าแมทเป็นแบรนด์ (SE/P###→SolarEdge, LP→Leapton, SPPM→Solar PPM, LR→Longi, Atrix→Sunwoda, BluE-S→KSTAR, ST→Schutten, HD→HUIDA, JKM→Jinko, JAM→JA Solar, TP→Talesun, EE2→Ekarat, "STP /SB "→SMA)
- **กันซ้ำด้วยเลข PI** (ตัด -R0/-R1 ออก) → export ไฟล์ใหม่แล้วนำเข้าทับ = อัปเดตดีลเดิม ไม่สร้างซ้ำ
- เก็บ piRef/piStatus/piDate ไว้ในโน้ตของโครงการ

## ช่องลูกค้า (autocomplete)
- พิมพ์ชื่อในช่อง "ลูกค้า / หน่วยงาน" → เลือกจากฐาน → เติม เบอร์/อีเมล/TaxID/ที่อยู่ ให้อัตโนมัติ
- ปุ่มสลับ **"ลูกค้าของฉัน" (ค่าเริ่มต้น) / "ทั้งหมด"** — ของคุณขึ้นก่อนเสมอ, ของเซลล์คนอื่นมีป้ายชื่อเซลล์กำกับ
- ข้อมูลลูกค้าจะถูกเก็บไปกับโครงการ (custTel/custEmail/custTaxId/custAddress) ใช้ทำใบเสนอราคาต่อได้
- ถ้าไม่มี `customers.js` แอปยังทำงานได้ปกติ แค่พิมพ์ชื่อลูกค้าเอง (ไม่มี autocomplete)
- อัปเดตฐานลูกค้า: รัน `python3 clean_customers.py` แล้วแปลงเป็น customers.js ใหม่

## วิธีใช้งานเบื้องต้น
1. เปิด `index.html` ด้วย Chrome / Safari
2. กด **+ เพิ่มโครงการ** กรอกข้อมูล → บันทึก
3. สลับมุมมองด้วยแท็บด้านบน:
   - **ภาพรวม** — KPI: จำนวนโครงการในมือ, มูลค่า Pipeline, มูลค่าถ่วงน้ำหนัก %, ปิดได้แล้ว + funnel ตามสถานะ + มูลค่าตามสินค้า
   - **ตาราง** — ค้นหา / กรอง / เรียงทุกคอลัมน์ คลิกแถวเพื่อแก้ไข
   - **Kanban** — เห็น pipeline เป็นภาพ ลากการ์ดข้ามคอลัมน์ หรือแตะปุ่ม ←/→ เพื่อเลื่อนสถานะ (ใช้บนมือถือได้)
4. กด **Excel** เพื่อ export เป็นไฟล์ .csv (เปิดด้วย Excel ได้)

ข้อมูลเก็บในเครื่อง (localStorage) ใช้งานได้แม้ไม่มีเน็ต

## สถานะโครงการ (Pipeline stages)
มุ่งหวัง → สำรวจ/ออกแบบ → เสนอราคา → เจรจา → ปิดได้ / หลุด
(เลื่อนสถานะแล้ว % โอกาสปิดจะปรับให้อัตโนมัติ แต่แก้เองได้)

## ดูข้ามเครื่อง (มือถือ + คอม) — workflow แนะนำ
1. **คอม:** กด "นำเข้า PI" จากไฟล์ Report_sale → ข้อมูลเข้าแอป + ซิงก์ขึ้น Google Sheet อัตโนมัติ
2. **มือถือ:** เปิดแอป (URL เดียวกัน) → ดึงจาก Sheet → เห็น pipeline เดียวกัน แก้/ดูได้
3. ทุกการแก้บนเครื่องไหนก็ซิงก์ขึ้น Sheet → อีกเครื่องกดเปิด/รีเฟรชก็เห็นล่าสุด
- ป้าย "📥 นำเข้า / ✏️ เพิ่มเอง" บอกที่มาของแต่ละดีล · หัวแอปบอก "อัปเดตล่าสุด"

> ⚠️ **ความเป็นส่วนตัว — ห้าม host `customers.js` ขึ้นเว็บสาธารณะ**
> `customers.js` มีลูกค้า 4,768 ราย พร้อมเบอร์/TaxID/ที่อยู่ (ข้อมูลส่วนบุคคล/PDPA)
> ถ้าจะ host บน GitHub Pages ให้อัปโหลด **เฉพาะ `index.html`** (แอปทำงานได้ไม่มี customers.js — แค่ autocomplete ลูกค้าจะว่าง)
> เก็บ `customers.js` ไว้ใช้บน **คอมส่วนตัว** เท่านั้น (เปิดไฟล์ในเครื่อง) เพื่อให้ได้ autocomplete ครบ
> ข้อมูล pipeline ทั้งหมดอยู่ใน Google Sheet (ส่วนตัวของคุณ) อยู่แล้ว

> `sales_history.js` มีชื่อลูกค้า เลขบิล สินค้า และยอดขาย จึงไม่ควรอัปโหลดขึ้นเว็บสาธารณะเช่นกัน
> หากต้องเปิดให้ผู้บริหารดู ควรใช้พื้นที่ที่จำกัดสิทธิ์เข้าถึงหรือเปิดจากเครื่อง/เครือข่ายภายใน

## ยอดขายย้อนหลัง
- หน้า **ภาพรวม** มีกราฟครบทุกปี 2559–2569 และเลื่อนแนวนอนได้เมื่อหน้าจอแคบ
- แท็บ **ยอดขาย** แสดงยอดขายสุทธิ จำนวนบิล จำนวนลูกค้า ยอดเฉลี่ยต่อบิล และยอดรายเดือน
- Top Customers สลับอันดับได้ตามยอดขายหรือจำนวนบิล
- ค้นหาลูกค้าจากข้อมูลทุกปี → ดูยอดสะสมและประวัติรายปี → กดบิลเพื่อดูสินค้า จำนวน และยอดขาย
- Top สินค้าสลับดูได้ตามแบรนด์ หมวดสินค้า หรือ SKU
- เมนูตั้งค่ามีปุ่มสำรองและกู้คืนข้อมูลแบบ JSON ซึ่งเก็บรายละเอียดโครงการ สินค้า และประวัติกิจกรรมครบถ้วน
- หน้า Dashboard มีศูนย์งานวันนี้ สถานะวันที่ข้อมูล และยอดขายจริงเทียบเป้ารายเดือน
- ช่องค้นหาหลักรองรับเลข PI และเปิดรายละเอียดพร้อมรายการสินค้าได้โดยตรง
- ปีปัจจุบันแสดงป้าย `YTD` เพื่อไม่ให้เปรียบเทียบกับปีเต็มโดยไม่ตั้งใจ
- **อัปเดตยอดขาย (แนะนำ):** กดปุ่ม **"นำเข้ายอดขาย"** บนแถบเครื่องมือ → เลือกไฟล์ `สรุปยอดขาย ….xlsx` → แอปอ่าน `.xlsx` ในเครื่อง (ไม่ส่งออกไปไหน ไม่ต้องใช้ไลบรารีภายนอก) จัดบิลเข้าปี พ.ศ. ตามวันที่ใบสำคัญ แล้วเก็บลง localStorage ทับข้อมูลเดิม — ไม่ต้องเปิด terminal
- ทางเลือกเดิม (ผ่าน terminal): เพิ่ม/แก้ชีตในไฟล์ Excel แล้วรัน `python3 scripts/build_sales_history.py` เพื่อสร้าง `sales_history.js` ใหม่ (ใช้เมื่ออยากบันเดิลข้อมูลไปกับ index.html)

## เชื่อมต่อ Google Sheet (ใช้บนมือถือ + สำรองข้อมูล)
ทำครั้งเดียว:
1. สร้าง Google Sheet ใหม่ → เมนู **ส่วนขยาย › Apps Script**
2. ลบโค้ดเดิม วางโค้ดจาก `AppsScript.gs` → บันทึก
3. **Deploy › New deployment › Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
4. คัดลอก Web app URL (ลงท้าย `/exec`)
5. ในแอปกดปุ่ม **เชื่อมต่อ Sheet** (มุมขวาบน) → วาง URL → บันทึก & ซิงก์

หลังจากนั้นทุกครั้งที่เพิ่ม/แก้/ลบ ข้อมูลจะซิงก์ขึ้น Sheet อัตโนมัติ
เปิดแอปจากเครื่องอื่น/มือถือ แล้วใส่ URL เดียวกัน → กด "ดึงจาก Sheet" เพื่อโหลดข้อมูล

> ถ้าอัปเดตจากเวอร์ชันเดิมและต้องการซิงก์ **ประวัติกิจกรรม (Activity History)** ข้ามเครื่อง
> ให้นำ `AppsScript.gs` เวอร์ชันล่าสุดไปวางทับ แล้ว Deploy › Manage deployments › New version หนึ่งครั้ง
